## 1. Overview

### 1.1 Purpose

This document defines the copy and content modifications required to customize the downloaded React template into the KiwiCar landing page. The landing page serves as a product showcase to explain features, value proposition, and build brand awareness before the main app launch.

### 1.2 Goals

| Goal | Description | Success Metric |
|------|-------------|----------------|
| Brand Introduction | Introduce KiwiCar as NZ's trusted used car platform | Brand recall in user surveys |
| Value Communication | Clearly communicate the 3 core value props | Time on page > 60s |
| Lead Generation | Capture interested users via waitlist signup | Email signup rate > 5% |
| SEO Foundation | Establish search presence for key terms | Indexed on Google within 2 weeks |

### 1.3 Target Audience

- **Primary:** NZ residents looking to buy/sell used cars
- **Secondary:** New migrants to NZ needing their first car
- **Tertiary:** Tech-savvy users frustrated with existing platforms

## 4. Design Guidelines

### 4.1 Brand Colors

| Usage | Color | Hex |
|-------|-------|-----|
| Primary | Kiwi Green | #22C55E |
| Secondary | Deep Blue | #1E40AF |
| Accent | Warm Orange | #F97316 |
| Background | Off-white | #FAFAFA |
| Text | Dark Gray | #1F2937 |

### 4.2 Typography

- **Headlines:** Bold, modern sans-serif (e.g., Inter, Poppins)
- **Body:** Clean, readable (e.g., Inter, system fonts)
- **Sizes:** H1: 48-56px, H2: 32-40px, Body: 16-18px

### 4.3 Imagery

- Use high-quality car photography (diverse vehicle types)
- Include people in lifestyle shots (diverse NZ demographics)
- UI mockups should match the upcoming app design
- Prefer images showing NZ context (landscapes, city backgrounds)

## 5. Technical Requirements

### 5.1 Performance

| Metric | Target |
|--------|--------|
| Lighthouse Performance | > 90 |
| First Contentful Paint | < 1.5s |
| Largest Contentful Paint | < 2.5s |
| Total Page Size | < 2MB |

### 5.2 Analytics Integration

- Google Analytics 4 (GA4)
- Track: Page views, scroll depth, CTA clicks, form submissions
- Conversion event: `waitlist_signup`

### 5.3 Form Integration

- Connect waitlist form to email service (e.g., Mailchimp, ConvertKit, or custom backend)
- Store submissions with: email, region, seller_interest, signup_date, utm_source
- Send confirmation email on signup

### 5.4 Responsive Breakpoints

| Breakpoint | Width |
|------------|-------|
| Mobile | < 640px |
| Tablet | 640px - 1024px |
| Desktop | > 1024px |
